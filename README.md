# WoVo

WoVo is a Tauri 2 + Leptos desktop app for monitoring Codex account usage, quota windows, account health, notifications, and account switching.

## Development

```sh
pnpm install
pnpm run tauri:dev
```

Useful checks:

```sh
pnpm run build:css
cargo fmt --all
cargo clippy --workspace --all-targets
cargo test --workspace
```

Build local production bundles with:

```sh
pnpm run tauri:build
```

## Releases

Releases are created by pushing a `vMAJOR.MINOR.PATCH` tag, for example `v0.1.0`. The tag version must match `package.json`, root `Cargo.toml`, `src-tauri/Cargo.toml`, and `src-tauri/tauri.conf.json`. GitHub Actions leaves the release as a draft for maintainer review.

Updater-enabled releases require these GitHub Actions repository secrets before the release workflow can publish artifacts:

- `TAURI_SIGNING_PRIVATE_KEY`: the one-line base64 private key generated by `pnpm exec tauri signer generate`.
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`: the password used when generating the private key, if the key is password protected.

The private key must match the updater `pubkey` in `src-tauri/tauri.conf.json`. To validate the secret material locally without building the app:

```sh
TAURI_SIGNING_PRIVATE_KEY="$(cat /path/to/updater-private.key)" \
TAURI_SIGNING_PRIVATE_KEY_PASSWORD="your-key-password" \
node scripts/validate-updater-signing-secrets.mjs
```

Set the GitHub secrets with:

```sh
gh secret set TAURI_SIGNING_PRIVATE_KEY --repo arvemy/wovo --body "$(cat /path/to/updater-private.key)"
gh secret set TAURI_SIGNING_PRIVATE_KEY_PASSWORD --repo arvemy/wovo --body "your-key-password"
```

If the original private key for the checked-in updater `pubkey` is unavailable, generate a new updater key pair, update the `pubkey`, and then set these secrets from the new private key.
